HDFS Notes

Posted on November 17, 2017

Based on HDFS Architecture. At the time of writing the version is Apache Hadoop 3.0.0-beta1.

Q: What’s HDFS?

A: The Hadoop Distributed File System (HDFS) is a distributed file system. It’s part of the core of Apache Hadoop.

Q: What is a Distributed File System?

A: An abstraction. It usually presents itself as a file system that we’re all familiar with, but under the hood files are stored distributedly on multiple hosts.

Q: Why do we want that?

A: Same reason why we do distributed computing - to deal with huge datasets that one single host will have trouble with.

Q: How do the hosts work together?

A: Typically there’s one NameNode and multiple DataNodes. The NameNode is sort of the master and it has all the metadata (how each file is splitted into blocks and where they are, etc.). It also serves as the point of contact for clients.

Q: Isn’t the NameNode a single point of failure then?

A: Yeah … but it also makes things simple. And there are ways to mitigate that but it complicates things. Trade-offs.

Q: How does HDFS achieve fault-tolerance?

A: By redundancy! Blocks are replicated and replicas of the same block are never stored on the same node.
