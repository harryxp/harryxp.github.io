Playing with Radare2

Posted on February 17, 2018

I’ve been playing with Radare2 and am very impressed with it.

Some tricks I’ve found useful:

  • Run it with r2 -A <binary>. -A does a lot for you automatically. To see the difference here’s one example where I ran it without -A but compensated it later by doing aaa:

      [harry@desktop:/home/harry 0] r2 <my-binary>
   -- You can debug a program from the graph view ('ag') using standard radare2 commands
  [0x08048430]> s @main
  0x80486b6
  [0x08048430]> pdf @main
  p: Cannot find function at 0x080486b6
  [0x08048430]> aaa
  [x] Analyze all flags starting with sym. and entry0 (aa)
  [x] Analyze len bytes of instructions for references (aar)
  [x] Analyze function calls (aac)
  [x] Use -AA or aaaa to perform additional experimental analysis.
  [x] Constructing a function name for fcn.* and sym.func.* functions (aan)
  [0x08048430]> pdf @main
              ;-- main:
  / (fcn) sym.main 58
  |   sym.main ();

  • Use the Visual Mode:

      [0x08048430]> V<enter>

  • In the Visual Mode, press p or P to rotate through panels. Details.

  • The Visual Mode has a lot of useful commands. I found jumping to functions particularly useful (o <function name> <enter>).

  • In the Visual Mode, pressing V gets you the graph view, and you get a nice little minimap. Again, use p or P to rotate.

  • In the Visual Mode, : gets you the command line, where you can use the online help mentioned below.

  • In the Visual Mode, c gets you the cursor mode. It comes in handy when you want to redefine bytes by d.

  • Online help is very useful. When in doubt, just bash ? like there’s no tomorrow. You can ask about subcommands by doing things like a? (:a? in the Visual Mode).

  • Extra fun:

    • Check out other tools that come with it: rax2, rasm2, etc.

Radare2’s UI is very reminiscent of Vim - you get a command line mode and a visual mode, although its visual mode is more like Vim’s normal mode. It also gets Vim-style keymap and even the idea of command composition.
